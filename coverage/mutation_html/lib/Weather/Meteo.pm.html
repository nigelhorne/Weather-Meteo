   39:

   45: =over 4
   46: 
   47: =item * Caching
   48: 
   49: Identical requests are cached (using L<CHI> or a user-supplied caching object),
   50: reducing the number of HTTP requests to the API and speeding up repeated queries.
   51: 
   52: This module leverages L<CHI> for caching geocoding responses.
   53: When a geocode request is made,
   54: a cache key is constructed from the request.
   55: If a cached response exists,
   56: it is returned immediately,
   57: avoiding unnecessary API calls.
   58: 
   59: =item * Rate-Limiting
   60: 
   61: A minimum interval between successive API calls can be enforced to ensure that the API is not overwhelmed and to comply with any request throttling requirements.
   62: 
   63: Rate-limiting is implemented using L<Time::HiRes>.
   64: A minimum interval between API
   65: calls can be specified via the C<min_interval> parameter in the constructor.
   66: Before making an API call,
   67: the module checks how much time has elapsed since the
   68: last request and,

  454: =head2 ua
  455: 
  456: Accessor method to get and set UserAgent object used internally.
  457: You can call I<env_proxy> for example, to get the proxy information from
  458: environment variables:
  459: 
  460:     $meteo->ua()->env_proxy(1);
  461: 
  462: You can also set your own User-Agent object:
  463: 
  464:     use LWP::UserAgent::Throttled;
  465: 
  466:     my $ua = LWP::UserAgent::Throttled->new();
  467:     $ua->throttle('open-meteo.com' => 1);
  468:     $meteo->ua($ua);
  469: 
  470: =head3 API specification
  471: 
  472: =head4 Input
  473: 
  474: When called with no arguments acts as a getter; the input schema is empty.
  475: When called with an argument the argument must be an object that responds to C<get>:
  476: 
  477:     { ua => { type => 'object', can => 'get' } }
  478: 
  479: =head4 Output
  480: 
  481:     { type => 'object', can => 'get' }
  482: 
  483: =head3 FORMAL SPECIFICATION
  484: 
  485:     ___ UA ____________________________________________________
  486:     | self?   : Weather::Meteo                               |
  487:     | ua?     : OBJECT [can 'get']   (optional)              |
  488:     |____________________________________________________________|
  489:     | result! : OBJECT [can 'get']                            |
  490:     |____________________________________________________________|
  491:     |                                                          |
  492:     | PRE ua? defined ^ ~ua?.can('get')                       |
  493:     |   => croak /must be an object that understands the get method/ |
  494:     |                                                          |
  495:     | POST ua? defined                                         |
  496:     |   => self?.ua = ua? ^ result! = ua?                     |
  497:     |                                                          |
  498:     | POST ~ua?                                                |
  499:     |   => result! = self?.ua  (no state change)              |
  500:     |____________________________________________________________|
