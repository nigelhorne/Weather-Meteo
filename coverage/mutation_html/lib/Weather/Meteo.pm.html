   62:

   64:

   67: 
   68: When a request is made,

   70: If a cached response exists it is returned immediately,
   71: avoiding unnecessary API calls.
   72: 
   73: =item * Rate-Limiting
   74: 
   75: A minimum interval between successive API calls can be enforced to ensure that the
   76: API is not overwhelmed and to comply with any request throttling requirements.
   77: 
   78: Rate-limiting is implemented using L<Time::HiRes>.
   79: A minimum interval between API calls can be specified via the C<min_interval> parameter
   80: in the constructor.

 1121: =head1 LIMITATIONS
 1122: 
 1123: =over 4
 1124: 
 1125: =item * Archive data lag
 1126: 
 1127: The Open-Meteo archive endpoint has a lag of approximately five days before
 1128: recent historical data becomes available.
 1129: For dates within the past five days,
 1130: C<weather()> may return C<undef> even when no error occurs.
 1131: Use C<forecast()> or C<sunrise_sunset()> (without a date) to obtain data for
 1132: today or recent days.
 1133: 
 1134: =item * Coordinate range
 1135: 
 1136: The module normalises coordinates with a bare leading decimal point (e.g.
 1137: C<".5"> to C<"0.5">) but does not validate that latitude is within C<-90..90>
 1138: or longitude within C<-180..180>.
 1139: Out-of-range values are passed to the API, which may return an error.
 1140: 
 1141: =item * No sub-hourly resolution
 1142: 
 1143: The hourly data arrays always contain exactly 24 entries per day (one per hour).
 1144: Sub-hourly resolution is not supported by this interface.
 1145: 
 1146: =item * Per-process rate limiting
 1147: 
 1148: The C<min_interval> rate limiter tracks the last request timestamp within a
 1149: single process instance.
 1150: Multiple concurrent processes or threads are not coordinated and may collectively
 1151: exceed the desired request rate.
 1152: 
 1153: =item * Timezone resolution requires an API key
 1154: 
 1155: Automatic per-location timezone resolution requires setting the
 1156: C<TIMEZONEDB_KEY> environment variable to a valid key from
 1157: L<https://timezonedb.com>.
 1158: Without it the module defaults to C<Europe/London> for all locations.
 1159: 
 1160: =item * No list-context support
 1161: 
 1162: C<weather()> and C<forecast()> enforce scalar/hashref context via
 1163: L<Return::Set>.
 1164: List context is not currently supported.
 1165: 
 1166: =item * Access control by convention only
 1167: 
 1168: Private methods (prefixed with C<_>) are not enforced by a module such as
 1169: L<Sub::Private>.
