   34: =head1 SYNOPSIS
   35: 
   36: The C<Weather::Meteo> module provides an interface to the Open-Meteo API for retrieving historical weather data from 1940.
   37: It allows users to fetch weather information by specifying latitude, longitude, and a date.
   38: The module supports object-oriented usage and allows customization of the HTTP user agent.
   39: 
   40:       use Weather::Meteo;
   41: 
   42:       my $meteo = Weather::Meteo->new();
   43:       my $weather = $meteo->weather({ latitude => 0.1, longitude => 0.2, date => '2022-12-25' });
   44: 
   45: =over 4
   46: 
   47: =item * Caching
   48: 
   49: Identical requests are cached (using L<CHI> or a user-supplied caching object),
   50: reducing the number of HTTP requests to the API and speeding up repeated queries.
   51: 
   52: This module leverages L<CHI> for caching geocoding responses.
   53: When a geocode request is made,
   54: a cache key is constructed from the request.
   55: If a cached response exists,
   56: it is returned immediately,
   57: avoiding unnecessary API calls.
   58: 
   59: =item * Rate-Limiting
   60: 
   61: A minimum interval between successive API calls can be enforced to ensure that the API is not overwhelmed and to comply with any request throttling requirements.
   62: 
   63: Rate-limiting is implemented using L<Time::HiRes>.
   64: A minimum interval between API
   65: calls can be specified via the C<min_interval> parameter in the constructor.
   66: Before making an API call,
   67: the module checks how much time has elapsed since the
   68: last request and,
   69: if necessary,
   70: sleeps for the remaining time.
   71: 
   72: =back

  236: The date argument can be an ISO-8601 formatted date,
  237: or an object that understands the strftime method.
  238: 
  239: Takes an optional argument, tz, containing the time zone.
  240: If not given, the module tries to work it out from the given location,
  241: for that to work set TIMEZONEDB_KEY to be your API key from L<https://timezonedb.com>.
  242: If all else fails, the module falls back to Europe/London.
  243: 
  244: Dates before 1940 return C<undef> silently.
  245: Invalid date strings cause a C<carp> and return C<undef>.
  246: Missing required arguments or non-numeric coordinates cause a C<croak>.
  247: 
  248: On success returns a hashref containing at minimum the key C<hourly>.
  249: Returns C<undef> if the API returns an error, if the JSON cannot be
  250: parsed, or if the response contains no C<hourly> key.
