   50:       my $forecast = $meteo->forecast({ latitude => 51.34, longitude => 1.42 });

   57: 
   58: =item * Caching
   59: 
   60: Identical requests are cached (using L<CHI> or a user-supplied caching object),
   61: reducing the number of HTTP requests to the API and speeding up repeated queries.
   62: 
   63: This module leverages L<CHI> for caching geocoding responses.
   64: When a geocode request is made,
   65: a cache key is constructed from the request.
   66: If a cached response exists,

   70: =item * Rate-Limiting

   73: 
   74: Rate-limiting is implemented using L<Time::HiRes>.
   75: A minimum interval between API

  596: Returns a hashref with C<sunrise> and C<sunset> ISO-8601 datetime strings
  597: (e.g. C<2022-12-25T08:09>) for the given location and date.
  598: 
  599: If no date is supplied, today is used and the forecast endpoint is queried.
  600: For historical dates (strictly before today) the archive endpoint is used.
  601: For today and future dates the forecast endpoint (L<https://api.open-meteo.com>) is used.
  602: 
  603: Takes an optional C<tz> argument for the time zone; defaults to C<Europe/London>.
  604: 
  605: Returns C<undef> if the API returns an error or if the response does not contain
  606: sunrise/sunset data.
