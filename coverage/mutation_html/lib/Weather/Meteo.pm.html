   45: The module supports object-oriented usage and allows customisation of the HTTP user agent.
   46:

   62: 
   63: =item * Caching
   64: 
   65: Identical requests are cached (using L<CHI> or a user-supplied caching object),
   66: reducing the number of HTTP requests to the API and speeding up repeated queries.
   67: 
   68: When a request is made,

   70: If a cached response exists it is returned immediately,
   71: avoiding unnecessary API calls.
   72: 
   73: =item * Rate-Limiting
   74: 
   75: A minimum interval between successive API calls can be enforced to ensure that the
   76: API is not overwhelmed and to comply with any request throttling requirements.
   77:

   79: A minimum interval between API calls can be specified via the C<min_interval> parameter
   80: in the constructor.

 1190: =head1 LIMITATIONS
 1191: 
 1192: =over 4
 1193: 
 1194: =item * Archive data lag
 1195: 
 1196: The Open-Meteo archive endpoint has a lag of approximately five days before
 1197: recent historical data becomes available.
 1198: For dates within the past five days,
 1199: C<weather()> may return C<undef> even when no error occurs.
 1200: Use C<forecast()> or C<sunrise_sunset()> (without a date) to obtain data for
 1201: today or recent days.
 1202: 
 1203: =item * Coordinate range
 1204: 
 1205: The module normalises coordinates with a bare leading decimal point (e.g.
 1206: C<".5"> to C<"0.5">) but does not validate that latitude is within C<-90..90>
 1207: or longitude within C<-180..180>.
 1208: Out-of-range values are passed to the API, which may return an error.
 1209: 
 1210: =item * No sub-hourly resolution
 1211: 
 1212: The hourly data arrays always contain exactly 24 entries per day (one per hour).
 1213: Sub-hourly resolution is not supported by this interface.
 1214: 
 1215: =item * Per-process rate limiting
 1216: 
 1217: The C<min_interval> rate limiter tracks the last request timestamp within a
 1218: single process instance.
 1219: Multiple concurrent processes or threads are not coordinated and may collectively
 1220: exceed the desired request rate.
 1221: 
 1222: =item * Timezone resolution requires an API key
 1223: 
 1224: Automatic per-location timezone resolution requires setting the
 1225: C<TIMEZONEDB_KEY> environment variable to a valid key from
 1226: L<https://timezonedb.com>.
 1227: Without it the module defaults to C<Europe/London> for all locations.
 1228: 
 1229: =item * No list-context support
 1230: 
 1231: C<weather()> and C<forecast()> enforce scalar/hashref context via
 1232: L<Return::Set>.
 1233: List context is not currently supported.
 1234: 
 1235: =item * Access control by convention only
 1236: 
 1237: Private methods (prefixed with C<_>) are not enforced by a module such as
 1238: L<Sub::Private>.
 1239: Callers are expected to treat them as internal; white-box test files may
 1240: access them directly.
 1241: 
 1242: =item * Host parameter restricted to plain DNS hostnames
 1243: 
 1244: The C<host> constructor parameter (and the C<WEATHER__METEO__host> environment
 1245: variable) must match C</\A[A-Za-z0-9][A-Za-z0-9.\-]*(:\d{1,5})?\z/>.
 1246: IP addresses in CIDR notation, URLs with path components, C<@>-style
 1247: user-info, and other special characters are rejected with a C<croak> to
 1248: prevent Server-Side Request Forgery.
 1249: If you need to test against a local service on a non-standard port, use a
